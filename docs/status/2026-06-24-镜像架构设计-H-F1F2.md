## 0. 设计目标

把当前"乱 + 有假跑bug"的镜像, 重写成一套**干净、可复现、GPU优先(CPU可fallback)**的端到端推理镜像, 锁定三源真实基线。**只做地基, 不在此版追分** —— 追分靠后续 cycle (e2v/orthofuse/阈值/加源)。

## 2. 目标目录结构 (推理镜像只需这些)

```
src/
  infer.py            ← 端到端主入口 (重写, 替代乱的 infer_full.py)
  sources/            ← 各源推理模块 (清晰分离)
    context.py        ← ctx LGBM (5 joblib + varlen featurize)
    ssl_encoder.py    ← whisper/hubert 统一 SSL encoder+head 推理
    fusion.py         ← per-class 融合 (基线: 简化加权; H-F4 还原 orthofuse)
  common.py           ← 常量(LAB/SUBMIT/COL2K/THR)+ 设备选择 + IO
models/               ← 权重 (COPY 进镜像的只有推理用到的)
  ctx_only/           ← lgbm_{c,t,bc,i,na}.joblib + thresholds.json + feature_spec.json
  whisper-large-v3/   ← base encoder
  chinese-hubert-large/
  wsp_head/model.pt   hub_head/model.pt
run.sh                ← 入口(官方要求/app/run.sh)
Dockerfile.finals     ← 唯一构建源
VERSIONS.md           ← tag 账本
```

> 不COPY: tools/ cloud/(训练代码, 推理不需要; 决赛阶段二审核单独提供训练代码包) / e2v/Omni权重(本版不用)。

## 3. 各源加载契约 (训练-推理一致性, 必须严格对齐)

> 来自 cloud/train_head_bcaug.py + extract_*_cuda.py。head 吃 `[2,80,FDIM]` 帧特征。

| 源 | encoder 加载 | extractor | FDIM | 帧池化 | head |
|---|---|---|---|---|---|
| ctx | — (LGBM joblib) | — | — | varlen featurize→46维 | 5×LGBM 逐类 |
| whisper | `WhisperModel.from_pretrained(...).encoder` | WhisperFeatureExtractor | 1280 | adaptive_avg_pool1d→80帧 | WhisperVAP(ctx_dim=46,wd=1280) |
| hubert | `HubertModel.from_pretrained(...)` | Wav2Vec2FeatureExtractor | 1024 | →80帧 | WhisperVAP(wd=1024) |
| ~~e2v~~ | ~~funasr.AutoModel(frame)~~ | ~~funasr~~ | ~~768~~ | — | H-F1b 独立修 |

- 音频: 8kHz双声道→取末8s→resample 16k→各声道独立提帧→`[2,80,FDIM]`。whisper走mel, hubert走input_values。
- WhisperVAP head 结构: proj+cross-attn(query)+ctx BatchNorm → concat → MLP→5 logits。(infer_full.py:148 已有, 复用)

## 4. GPU优先 / CPU fallback (v2核心成果, 保留为一等公民)

```python
DEV = "cuda" if torch.cuda.is_available() else "cpu"   # 评测有GPU→走cuda
DTYPE = torch.float16 if DEV == "cuda" else torch.float32
# 各 encoder/head .to(DEV); per-file try-except 继续(单段失败不崩全局, 出safe default)
```
- 评测环境有GPU(官方baseline支持CUDA) → 正常走 cuda fp16。
- CPU fallback 保留(本机调试/评测GPU异常兜底), 但耗时上界大(MPS 18.75s/段)。
- **60min约束**: 镜像构建后必须GPU实测1000段(whisper-large-v3是大头), 私有测试集规模未知按最坏估。

## 5. 融合 (基线版, 简化; H-F4 还原 orthofuse)

- **基线(本版)**: 沿用初赛 orthofuse **离线选定的 per-class 策略**(不是在线重选)。
  - 初赛 orthofuse = OOF 上 per-class 五选一(ctx/whisper/eq/w70/w30 固定权重), 选定结果固定。
  - 本版从初赛产物取 S5 每类选定策略写死进 fusion.py。
- 三源版先验证 ctx×wsp×hub per-class 融合跑通, 数值合理。
- H-F4 再精确还原 orthofuse 完整 per-class 配方(对照初赛真SOTA)。

## 6. 阈值 (本版用初赛验证值, 不用当前偏离的硬编码)

- 当前 infer_full.py 硬编码 `THR=[0.05,0.50,0.75,0.65,0.25]`(C,T,BC,I,NA)偏离初赛。
- 初赛 `ctx_only/thresholds.json` = {c:0.05, t:0.5, bc:0.5, i:0.5, na:0.05}。
- **本版用 thresholds.json**(读文件, 不硬编码), 还原初赛验证过的阈值。NA 0.05(非0.25)关键: NA是30%大类, 高阈值砍崩(违阈值铁律)。
- (注: 初赛阈值是 ctx-only 调的, 融合后最优阈值可能微调 → H-F5 用30s切片验证集精调, 不在滑窗CV激进搜)

## 7. Dockerfile.finals (唯一源, 修订要点)

```dockerfile
FROM finvcup-registry.cn-shanghai.cr.aliyuncs.com/finvcup/team35:finv_baseline
ENV PYTHONUNBUFFERED=1 OMP_NUM_THREADS=4
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends libsndfile1 libgomp1 && rm -rf /var/lib/apt/lists/*
RUN pip install --no-cache-dir lightgbm joblib scikit-learn pynvml   # +pynvml(官方要求), 三源不需peft/funasr
# 模型每源一层(缓存复用)
COPY models/ctx_only/ /app/models/ctx_only/
COPY models/whisper-large-v3/ /app/models/whisper-large-v3/
COPY models/chinese-hubert-large/ /app/models/chinese-hubert-large/
COPY models/wsp_head/ /app/models/wsp_head/
COPY models/hub_head/ /app/models/hub_head/
# 代码(最后, 改动频繁) — 只COPY src/ + run.sh, 不带tools/cloud
COPY src/ /app/src/
COPY run.sh /app/run.sh
RUN mkdir -p /app/submit && chmod +x /app/run.sh
CMD ["bash"]
```
- 三源镜像比四源+Omni小很多(去Omni 11G + e2v 360M)。
