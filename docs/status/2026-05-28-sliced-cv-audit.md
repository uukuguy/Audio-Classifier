# 30s 切片化验证集审计 — 修 CV 不可信漏洞

> 2026-05-28。脚本 `tools/climb/sliced_cv.py`。锚:cycle1 context-only 线上 0.710789。
> 结论:**cap1 切片 CV 是可信验证协议**(gap +0.118→+0.055),BC 是真瓶颈非 CV 假象。

## 问题

`cycle_context.py` 的 valid 用 stride=5 **密集滑窗**(整通 25min 对话切成几十万高度重叠窗),
但 test 是 **1000 个独立 30s 切片**(context 恒 375 chunk)。分布错配 → 滑窗 CV 不可信:
线上 0.7108 − 滑窗 CV 0.5923 = **gap +0.1185**。所有滑窗 CV 上调出的阈值/特征结论都站在
错配分布上(阈值铁律 −0.027 教训即此根因)。

## 方法

会话级 5-fold OOF。每折 train 对话密集滑窗训 LGBM,held-out 对话切**不重叠** 375+25 片段做 valid。
三档采样密度对照(同一批 OOF 模型,只是 valid 子采样不同):

- **cap1**:每通对话取第 1 个片段(369 valid,最像 test 的"每通≈1 独立片段")
- **cap5**:每通前 5 个片段(1845 valid)
- **all**:全部不重叠片段(18112 valid)

## 结果(5-fold)

| 协议 | n_val | Macro CV | gap (线上−CV) | C | NA | T | I | BC |
|---|---|---|---|---|---|---|---|---|
| 滑窗(旧) | 282k | 0.5923 | **+0.1185** | 0.971 | 0.796 | 0.539 | 0.437 | 0.219 |
| **切片 cap1** | 369 | **0.6561** | **+0.0547** | 0.974 | 0.871 | 0.633 | 0.581 | 0.222 |
| 切片 cap5 | 1845 | 0.6015 | +0.1093 | 0.974 | 0.828 | 0.608 | 0.448 | 0.149 |
| 切片 all | 18112 | 0.5948 | +0.1160 | 0.972 | 0.796 | 0.550 | 0.442 | 0.215 |

**cap1 推荐阈值**:C=0.05 / NA=0.35 / T=0.55 / I=0.55 / BC=0.55(均接近 0.5,符合阈值铁律)。

## 三个定论

1. **cap1 是可信 CV 协议**:gap 从 +0.118 砍到 +0.055。采样密度单调影响 gap(cap1<cap5<all≈滑窗)——
   越密集采样越偏离 test 独立片段分布,CV 越悲观。**今后所有本地实验用 cap1 切片 CV 判优劣,不用滑窗。**
2. **线上"超额"来自 NA/T/I**:滑窗系统性低估这三类(NA 0.796→0.871,T 0.539→0.633,I 0.437→0.581)。
   独立片段分布下它们的可预测性远高于滑窗估计。0.7108 的高分主要是这三类贡献。
3. **BC 是真瓶颈,非 CV 假象**:三档采样 BC 全钉在 ~0.22(cap1=0.222/all=0.215),唯一不随验证协议改善的类。
   **确认 BC 需端到端神经编码器(whisper 语义/韵律),手工特征+LGBM 到顶。** 这是给上云路线的关键情报。

## 对上云的增益估算

cap1 可信 CV=0.656,BC F1=0.22。BC 等权占 Macro 的 1/5。若 whisper 把 BC 拉到 0.40
(research 认为可行),单 BC 贡献 ≈ (0.40−0.22)/5 = **+0.036 线上** → ~0.747(进前 3 ≥0.7357)。
**值得的模型赌注。**

## 复用

```bash
# 可信 CV(三档对照),今后判任何 context/特征改动用这个不用滑窗
OMP_NUM_THREADS=4 MKL_NUM_THREADS=4 VECLIB_MAXIMUM_THREADS=4 OPENBLAS_NUM_THREADS=4 \
  python3 tools/climb/sliced_cv.py --folds 5 --slice-stride 400
```
