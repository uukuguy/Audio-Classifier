## 4. S5 配方组件 × 端到端可行性 (核心决策表)

> 初赛 S5 = R4 + omni3b_ms2 0.05;  R4 = (ctx 5seed × wsp_ms × hub_ms per-class) + e2v_ms + hub_ms

| 组件 | 初赛多样性 | 本地权重 | 端到端代价 | 60min 可行性 | 复赛取舍 |
|---|---|---|---|---|---|
| **ctx LGBM** | variant-F **5 seed** | 仅单 seed (`ctx_only/`) | LGBM CPU 极快, 5seed 无压力 | ✅ 轻松 | **搬全 5 seed**(缺 seed1/7, 需重训或拉备份) |
| **whisper head** | **3 seed** mean | 仅单 `wsp_head/model.pt` | whisper enc **0.8-1.6s/段**(MPS), GPU 快但仍是大头 | ⚠️ 3 seed = enc 跑 3 遍? 或 enc 1 遍 3 head | enc 跑 1 遍 → 3 个 head 各推(head 极轻), **可行** |
| **hubert head** | 3 seed mean | 仅单 `hub_head/model.pt` | hubert enc 比 whisper 轻 | ✅ 同上 | enc 1 遍 + 3 head |
| **e2v head** | 3 seed mean | 仅单 `e2v_head/model.pt` | e2v enc 轻 | ✅ | enc 1 遍 + 3 head |
| **Omni-3B LoRA** | 3 seed × 5 fold | ✅ `omni3b-lora/seed{1,42,7}/fold{0-4}.pt` 全在(390M) | **11G 基座 + LLM 前向, CPU 必 OOM/超时** | 🔴 GPU 才行, +0.0004 vs R4 | **评测期默认砍**(收益微 + 超时高危); 仅 GPU 确认有余量再加 |

**多样性复现关键技巧**: SSL multi-seed 不需要 encoder 跑 N 遍 — encoder 前向 1 次得帧特征, 喂给 N 个 head(head 是轻量 MLP/attention, 毫秒级)取平均。所以 3 seed 的边际成本 ≈ 0。**真正的耗时瓶颈是 encoder 前向次数, 不是 seed 数**。

## 7. 推荐路线 (评测期 6/24-7/9, ~30 次提交配额)

> 原则: 先把管道组织干净 + 锁定可复现基线, 再逐路加多样性。每版换 tag, 每版留源码, 每版只动一个变量(便于归因, 呼应 climb 实验铁律)。

**Phase A — 工程地基(不改配方, 0 风险, 先做)**
1. 修 pynvml + 建 VERSIONS.md + 定 tag 规范(如 `r4-base` / `r4-5seed` / `s5-omni`)
2. 收敛 Dockerfile 到唯一源, 删冗余脚本, 瘦身 COPY
3. 融合权重 build-arg → ENV
4. 本地用 baseline `test_data` 跑通当前镜像, 实测端到端耗时(距 60min 多远)+ 各 encoder 正常

**Phase B — 锁定可复现基线**
5. 当前 R4 简配重新提交一版(干净 tag)→ 确认线上分(应 ≈ R4 端到端真实水平), 作为后续对照基线

**Phase C — 逐路加多样性(每版一个变量)**
6. ctx 单→5 seed (云端补 seed1/7, 或先本地能凑几个 seed) → 提交看增益
7. SSL head 单→3 seed (encoder 1 遍 + 3 head mean) → 提交看增益
8. (条件性) GPU 余量确认充足 → 加 Omni, 否则评测期不加

**Phase D — 决赛材料(并行, 7/15 截止)**
9. 保证 `cloud/train_*.py` 可跑通 + README(训练→推理全流程) + 技术报告 3 页

## 8. 不要再走的弯路 (ruled out)

- ❌ 把 `probs.npz` 搬进镜像 — 私有测试集必须现场重算, npz 无用
- ❌ 追求端到端 100% 复现初赛 0.747 — 初赛多样性(尤其 Omni 5fold×3seed)在 60min/CPU 下搬不全, 目标应是"约束内最优", 不是"复刻离线分"
- ❌ 动 GPU/CPU fallback 链路 — v2 硬骨头, 别碰
- ❌ 同 tag 推不同镜像 / 不留源码就推 (RESUME 已记)
