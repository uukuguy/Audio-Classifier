**【备1】评价指标：**

F1 值是机器学习中用于评估分类模型性能的核心指标，它综合了精确率（Precision, P）和召回率（Recall, R）两个指标，是精确率和召回率的调和平均：

F1 = 2*P*R/（P+R）

精确率是预测为正例的样本中，真正为正例的比例。召回率是所有真正例中，被正确预测为正例的比例。

P = TP/（TP+FP）

R = TP/（TP+FN）

其中，

➢TP（True Positive）：被正确预测的正例。即该数据的真实值为正例，预测值也为正例的情况；

➢FP（False Positive）：被错误预测的正例。即该数据的真实值为反例，但被错误预测成了正例的情况；

➢FN（False Negative）：被错误预测的反例。即该数据的真实值为正例，但被错误预测成了反例的情况。

 **1. 是否允许使用公开数据或公开模型？**

本次比赛允许参赛选手使用其他公开的语音、文本等相关的数据及模型，但参赛选手需要在**【2026年06月10日】**之前向主办方报备**（报备官方邮箱：xinyebei@xinye.com）**，注明使用的公开数据、下载地址及公开模型，届时我们会公布在网页上以供其他选手参考。



 **2. 测试集与训练集有什么区别？为什么线下训练得分很高，官网提交得分很低？**

● 本届比赛聚焦于“对话轮次交互建模”，训练集采取随机切分方式（选手可以任意构建切分）、公开测试集采取不重叠的随机切分、私有测试集采取不重叠的随机切分并包含内部私有数据。

● 公开测试集打分结果与私有测试集打分趋势仍有可能不同。现在在初赛打分比较好的选手，复赛成绩并不一定是最好的。



 **3. 可以使用公开测试集中的数据进行训练、生成、打标签吗？**

不可以利用公开测试集的数据进行训练、标注、生成数据，不可以以任何形式对公开测试集打伪标签。



 **4. 如果使用了数据增强，生成的额外数据怎么提交？**

请选手将生成的数据、生成数据的代码一并放入docker镜像中。



 **5. 复赛提交之后会实时公布成绩吗？**

因提交后需要进行预测打分，选手提交大概延时一段时间（估计在1小时内）可以看到成绩。

复赛提交窗口期（6月20日-7月7日）内，每天都有2次提交机会，大赛组委会取选手们最后一次提交为准。我们要求选手提交的镜像是可以直接运行训练、推理代码，如在运行过程中出现问题，组委会会与选手取得联系。



**7. 所有截止时间以什么时间为准？**

所有截止时间均为当日北京时间23:59:59。



 **8. 在复赛阶段是否可以提供运行的基础镜像？**
